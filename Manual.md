# VaseGeneratorWeb 한글 매뉴얼

VaseGeneratorWeb은 회전형 프로파일을 기반으로 콘크리트 압출용 꽃병 형상과 연속 카르테시안 툴패스를 생성하는 브라우저 애플리케이션입니다. 기본 장비 조건은 노즐 구경 35 mm, 레이어 높이 20 mm입니다.

> **안전 주의:** 생성된 G-code를 장비에 바로 투입하지 마십시오. 실제 출력 전에 장비 작업 영역, 좌표 원점, 이동 방향, 펌프 명령, 속도, 재료 배합, 충돌 가능성을 반드시 검증하고 소형 시험 출력을 수행해야 합니다.

## 1. 실행 방법

Node.js 22.13 이상이 필요합니다.

```bash
npm install
npm run dev
```

터미널에 표시되는 로컬 주소를 브라우저에서 엽니다. 배포된 버전은 다음 주소에서도 사용할 수 있습니다.

https://praxisdesign.github.io/VaseGeneratorWeb/

## 2. 화면 구성

- **왼쪽 패널:** 프로파일 프리셋, 프로파일 편집기, 출력 가능성 정보
- **중앙 뷰포트:** 3D 모델 미리보기와 출력 시뮬레이션
- **오른쪽 패널:** 전체 크기, 출력 속도, 안전 기준, 표면 패턴과 높이별 강도 설정
- **하단 시뮬레이션 패널:** 재생, 레이어 이동, 속도, G-code 줄 이동 및 출력 정보

마우스 또는 터치로 3D 화면을 회전·이동·확대/축소할 수 있습니다.

## 3. 기본 작업 순서

1. **Profile presets**에서 기본 형상을 선택합니다.
2. **Profile + heatmap** 그래프의 점을 드래그해 실루엣을 조정합니다.
3. **Dimensions**에서 크기와 출력 조건을 설정합니다.
4. 필요하면 **Surface pattern**에서 패턴과 구간별 강도를 설정합니다.
5. 3D 뷰를 **Material**, **Bead**, **Heatmap** 모드로 확인합니다.
6. 출력 시뮬레이션으로 툴패스 순서와 위험 구간을 점검합니다.
7. 검증이 끝나면 **Download G-code**를 눌러 파일을 저장합니다.

## 4. 프로파일 만들기

### 프리셋

다음 8가지 기본 프로파일을 제공합니다.

- Cylinder: 원통형
- Taper: 위로 갈수록 좁아지는 형상
- Classic: 몸통이 넓고 입구가 좁은 전통형
- Bottle: 병 형태
- Bulb: 하단이 둥글게 팽창한 형태
- Hourglass: 중앙이 잘록한 형태
- Flare: 상단이 넓어지는 형태
- Wave: 높이에 따라 굴곡이 반복되는 형태

### 프로파일 편집

**Profile + heatmap** 그래프에서 제어점을 드래그합니다.

- 좌우 이동: 해당 높이의 반지름 변경
- 상하 이동: 제어점의 높이 변경
- 프로파일 변경 시 3D 형상, 툴패스 및 출력 가능성 분석이 즉시 갱신됩니다.
- **Reset profile**은 현재 선택된 프리셋의 초기 프로파일로 되돌립니다.

## 5. 치수와 출력 조건

**Dimensions** 패널의 항목은 다음과 같습니다.

- **Overall scale:** 높이와 지름을 함께 25~200%로 조절
- **Height:** 기준 높이 200~1200 mm
- **Max diameter:** 기준 최대 지름 140~800 mm
- **Print speed:** 출력 이동 속도 300~3600 mm/min
- **Max layer offset:** 인접 레이어 사이에서 허용할 최대 수평 이동량 2~15 mm

실제 최종 높이와 지름에는 **Overall scale**이 적용됩니다. 크기를 변경하면 기존 시뮬레이션 위치가 초기화됩니다.

## 6. 표면 패턴

**Surface pattern**에서 다음 패턴을 선택할 수 있습니다.

- None: 패턴 없음
- Vertical Ribs: 세로 리브
- Horizontal Bands: 가로 밴드
- Diamond: 다이아몬드
- Spiral: 나선
- Cellular: 셀 형태
- Noise: 불규칙 요철

공통 설정:

- **Amplitude:** 표면 요철 깊이
- **Frequency:** 패턴 반복 횟수 또는 밀도
- **Twist:** 높이 방향 회전량
- **Seed:** Noise 패턴의 무작위 형태 변경

### 높이별 패턴 강도

패턴 아래 그래프의 제어점을 드래그해 높이별 강도를 설정합니다. 왼쪽은 하단, 오른쪽은 상단이며 점을 위로 올릴수록 패턴이 강해집니다. 기본값은 하단에서 패턴을 억제하고 중앙에서 강하게 만든 뒤 상단에서 감소하도록 구성되어 있습니다.

## 7. 미리보기 모드

중앙 상단의 탭으로 표시 방식을 바꿉니다. 모드를 바꿔도 현재 시뮬레이션 위치는 유지됩니다.

- **Material:** 완성된 재료 표면 형상
- **Bead:** 실제 적층 비드 형태의 툴패스
- **Heatmap:** 비드에 출력 위험도를 색상으로 표시

Heatmap에서 녹색은 안전한 구간, 빨간색에 가까울수록 설정한 최대 레이어 오프셋을 초과할 가능성이 큰 구간입니다.

## 8. 출력 가능성 히트맵

분석은 인접 레이어 사이의 수평 이동량을 기준으로 합니다.

- 상단 상태 표시가 **Within offset limit**이면 모든 분석 지점이 설정값 이내입니다.
- **risky points**가 표시되면 허용값을 초과한 지점이 존재합니다.
- **Maximum overhang** 카드에는 최대 수평 이동량, 추정 각도 및 높이(Z)가 표시됩니다.
- Maximum overhang 카드를 누르면 시뮬레이션이 해당 위험 지점으로 이동합니다.

히트맵은 기하학적 보조 지표이며 재료 점도, 경화 시간, 펌프 압력, 출력 속도 및 구조 좌굴은 계산하지 않습니다.

## 9. 출력 시뮬레이션

중앙 하단 컨트롤의 기능은 다음과 같습니다.

- 원형 화살표: 시뮬레이션 처음으로 이동
- 이전/다음 버튼: 한 레이어씩 이동
- 재생/일시정지: G-code 순서대로 노즐 마커 이동
- 속도 선택: 0.25×, 0.5×, 1×, 2×, 5×, 10×
- 타임라인: 원하는 툴패스 위치로 직접 이동
- **Line:** 특정 G-code 줄 번호로 이동
- **Layer / Z / Time:** 현재 레이어, 높이 및 예상 경과 시간 표시

설계 치수, 프로파일 또는 패턴을 바꾸면 툴패스가 다시 계산되므로 시뮬레이션은 처음으로 돌아갑니다.

## 10. G-code 내보내기

**Download G-code**를 누르면 `vase-toolpath.gcode` 파일이 저장됩니다. 현재 출력은 다음 특성을 가집니다.

- 단위: mm (`G21`)
- 좌표: 절대 카르테시안 좌표 (`G90`)
- 이동: 연속 `G1 X Y Z F` 명령
- 출력 시작/종료: `M3` / `M5`
- 기본 노즐 구경: 35 mm
- 기본 레이어 높이: 20 mm

장비가 다른 펌프 명령을 사용하거나 원점 설정 방식이 다르면 G-code를 장비 사양에 맞게 수정해야 합니다. 특히 현재의 `G92 X0 Y0 Z0`, `M3`, `M5`, 종료 시 `G0 Z100.000` 명령이 실제 장비에 안전한지 확인하십시오.

## 11. 권장 출력 전 점검표

- 모델 크기가 프린터 작업 영역 안에 있는가
- 꽃병 중심과 프린터 원점의 관계가 올바른가
- 노즐 구경과 레이어 높이가 실제 장비 설정과 일치하는가
- 최대 레이어 오프셋과 히트맵 위험 구간을 확인했는가
- 하단이 충분히 넓고 안정적인가
- 패턴 진폭과 Twist가 과도하지 않은가
- 이동 속도와 재료 토출량이 일치하는가
- 펌프 시작·정지 명령이 컨트롤러와 호환되는가
- 전체 G-code를 시뮬레이션하고 시험 출력했는가

## 12. 문제 해결

- **형상이 너무 크거나 작음:** Overall scale, Height, Max diameter를 확인합니다.
- **상단 또는 곡면이 빨갛게 표시됨:** 프로파일 경사를 완만하게 하거나 Max layer offset 기준과 실제 재료 성능을 재검토합니다.
- **패턴이 보이지 않음:** Pattern이 None인지, Amplitude가 0인지, 높이별 강도 그래프가 하단에 있는지 확인합니다.
- **Twist 변화가 약함:** 패턴을 선택하고 Frequency와 Amplitude를 높인 뒤 Twist를 조절합니다.
- **시뮬레이션이 처음으로 돌아감:** 형상 또는 툴패스 관련 설정 변경 시 정상 동작입니다.
- **G-code가 장비에서 실행되지 않음:** 컨트롤러가 사용하는 좌표계, 펌프 명령 및 G-code 문법을 장비 제조사 문서와 대조합니다.
